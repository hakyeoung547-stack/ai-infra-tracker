## 배운 핵심 개념

- **쉘** — 나와 OS 사이의 비서(통역사). 터미널에 명령 입력 → 쉘이 OS에 전달
- **`source`** — 쉘 메모장(현재 쉘 환경)을 직접 변경. 효과가 현재 쉘에 지속됨
- **`python`** — 새 프로세스(직원)를 불러서 일 시킴. 끝나면 퇴근, 쉘 환경 변화 없음
- **순서** — `source` 먼저 → `python` 실행 (메모 쓰고 나서 직원 부르기)
- **`echo`** — 터미널에 출력. `echo "msg" >> file` 은 파일에 로그 추가
- **`>>` vs `>`** — `>>` 파일에 이어서 추가 / `>` 파일 덮어쓰기
- **`$변수`** — 저장할 때 `VAR="값"` ($ 없음), 꺼낼 때 `"$VAR"` ($ 붙임)
- **`cat`** — 파일 전체 출력 (파일이 짧을 때)
- **`tail`** — 마지막 10줄 출력 / `tail -f` 실시간 모니터링 (cron 로그 확인용)

| 명령어 | 비유 | 언제 쓰나 |
|--------|------|-----------|
| `cat cron.log` | 책 처음부터 끝까지 읽기 | 파일이 짧을 때 |
| `tail cron.log` | 마지막 10줄만 보기 | 최근 로그 확인 |
| `tail -f cron.log` | 실시간으로 쓰여지는 책 보기 | 실행 중 모니터링 |
