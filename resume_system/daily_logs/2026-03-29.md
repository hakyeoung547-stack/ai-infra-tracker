## 어려웠던 점 / 해결 방법
- import할 때도 파일 안의 실행 코드가 전부 튀어나와버리는 문제
  - 원인: `fetch_all_prices()` 같은 함수 호출이 파일 최상단에 있으면 import 시 즉시 실행됨
  - 해결: `if __name__ == "__main__":` 블록으로 실행 코드를 감싸면 직접 실행할 때만 동작

## 배운 핵심 개념

- **파이썬 파일 실행 방식 2가지**

| 실행 방식 | `__name__` 값 | 실행되나? |
|-----------|--------------|-----------|
| `python fetch_prices.py` 직접 실행 | `"__main__"` | ✅ 실행됨 |
| 다른 파일이 `import` | 파일 이름 (`"fetch_prices"`) | ❌ 실행 안 됨 |

- **함수 정의 vs 함수 호출**
  - 함수 정의 (`def fetch_all_prices():`) → import해도 그냥 등록만 됨, 실행 안 됨
  - 함수 호출 (`fetch_all_prices()`) → `if __name__ == "__main__":` 안에 넣어야 import 시 실행 안 됨

- **`if __name__ == "__main__":` = 실행 코드 울타리**
  - "직접 실행할 때만 여기 아래 코드 돌려줘, import됐을 때는 건드리지 마"

```python
def fetch_all_prices():       # 함수 정의 → import해도 등록만 됨
    print("수집 중...")

if __name__ == "__main__":
    fetch_all_prices()        # 직접 실행할 때만 돌아감
```

## requirements.txt

### 역할
프로젝트에 필요한 패키지와 버전을 기록해두는 **장보기 목록**.
GitHub에 같이 올려두면 누가 클론해도 한 번에 환경 세팅 가능.

### 만드는 법
```bash
pip freeze > requirements.txt
```
- `pip freeze` — 현재 가상환경에 설치된 패키지 목록을 버전과 함께 출력
- `>` — 화면 출력 대신 파일로 방향을 꺾어 저장 (덮어쓰기)

```
numpy==2.4.3
pandas==3.0.1
yfinance==1.2.0
...
```

### 쓰는 법
```bash
pip install -r requirements.txt
```
- `-r` — read(읽기). 파일을 읽어서 목록에 있는 패키지를 전부 한 번에 설치

### 새 환경 세팅 순서
```bash
git clone ...                    # 코드 받고
python -m venv .venv             # 가상환경 만들고
source .venv/bin/activate        # 가상환경 켜고
pip install -r requirements.txt  # 패키지 한방에 설치
```
