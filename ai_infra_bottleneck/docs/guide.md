### 1단계 — 이 프로젝트는 무슨 질문에 답하려는가

**핵심 질문: "지금 AI 인프라 섹터 중 어디에 돈이 몰리고 있는가? 그리고 왜인가?"**

AI 데이터센터를 짓는 데는 GPU, 전력, 냉각, 네트워크가 전부 필요하다.
이 중 공급이 수요를 따라가지 못하는 구간의 기업은 가격 협상력이 생기고 주가가 먼저 움직인다.

이 시스템은 **주가 데이터로 그 신호를 매주 탐지**하고,
**어닝콜로 실물을 확인**하며, **인프라 맵으로 전체 구조를 시각화**한다.

### 3단계 — 그 데이터로 어떤 점수를 계산하는가

**병목 점수 v1 (현재 구현)**

```
병목 점수 = 이 섹터 3개월 수익률 − 전체 AI인프라 13개 기업 평균 3개월 수익률
```

예시: power_equipment 섹터(ETN+SMCI)가 3개월 +28%, 전체 평균이 +16% → 점수 = **+12%**

| 점수 | 의미 |
|------|------|
| +10% 초과 | 강한 병목 신호 가능성 |
| +5% ~ +10% | 주목 시작 |
| -5% ~ +5% | 중립 |
| -5% 미만 | 병목 해소 또는 약세 |

이 점수 하나로 바로 판단하면 안 된다. 다음 단계로 넘어간다.

### 4단계 — 그 점수를 바로 믿지 않고 무슨 체크를 더 하는가

| 질문 | 확인 방법 | 자동화 여부 |
|------|-----------|-------------|
| Q1. 점수가 오르는 추세인가? | 지난 4주간 점수 변화 확인 | ✅ 자동 |
| Q2. 처음 임계값을 넘은 건가? | +5%를 이번 주 처음 넘었는가 | ✅ 자동 |
| Q3. 이 섹터만 올랐나? | SPY(S&P500)와 비교 | ✅ 자동 |

4가지를 조합하면 8가지 결론 중 하나가 나온다:
🔴 강한 병목 신호 / 🟡 매크로 랠리 주의 / ⛔ 고점 경계 / 🟠 모니터링 강화 / 🟢 선취매 검토 / ⚪ 중립 / 🔵 섹터 교체 / 🟢 역발상 구간

## 이 시스템이 해결하려는 문제

투자 분석을 할 때 가장 흔히 빠지는 함정이 있다.

**함정 1 — 뉴스를 보고 산다**
NVIDIA가 역대 최고 실적을 발표했다. 주가는 그날 5% 올랐다. 사야 할까?
이미 늦었다. 시장은 실적 발표 6개월 전부터 알고 있었다.

**함정 2 — 점수 하나로 판단한다**
"이 섹터 3개월 수익률이 +15%다. 강하게 오르고 있으니 사자."
그런데 그 +15%의 절반은 SPY 전체 상승이었다. 섹터 고유 신호가 아니었다.

**함정 3 — 실물을 안 본다**
주가 차트만 본다. 그런데 주가는 결국 실물을 따라간다.
실물을 먼저 보면 주가가 움직이기 전에 포지션을 잡을 수 있다.

**이 시스템이 하려는 것**: 세 함정을 피하기 위해
가격 신호 + 신호 품질 체크 + 실물 근거를 결합한 구조화된 판단 체계를 만드는 것이다.

## 핵심 개념: 병목이란 무엇인가

AI 산업을 하나의 파이프라인으로 보자.

```
[AI 서비스 수요]
      ↓
[GPU 칩 설계 — NVIDIA, AMD]
      ↓
[반도체 제조 — TSMC, 삼성]
      ↓
[서버 조립 — SMCI]
      ↓
[데이터센터 전력 공급 — Eaton, Vertiv]
      ↓
[냉각 시스템 — VRT, Modine]
      ↓
[전력 인프라 — NEE, CEG, 송전망]
```

이 파이프라인에서 **전체 속도를 결정하는 건 가장 느린 구간**이다.

GPU를 1만 개 만들어도 데이터센터 전력이 안 들어오면 쓸 수 없다.
전력이 들어와도 냉각이 안 되면 서버가 꺼진다.

**가장 느린 구간 = 병목**이고, 이 병목 구간의 기업이 가격 협상력을 갖고 수혜를 받는다.

## 이 프로젝트에서 분석하는 섹터

### Phase 1 — 현재 분석 중 (4개 섹터, 13개 기업)

| 섹터 | 설명 | 포함 기업 |
|------|------|-----------|
| **semiconductor** | GPU 설계 + 파운드리 + 반도체 장비 | NVDA, AMD, TSM, AMAT, LRCX |
| **power_equipment** | 데이터센터용 전력 관리 장비, 서버 전력 | ETN (Eaton), SMCI |
| **power_utility** | 발전 + 전력 공급 + 송전망 시공 | NEE, CEG, ETR, PWR |
| **cooling** | 액체냉각 + 열관리 부품 | VRT (Vertiv), MOD |

병목 점수를 계산할 때 "섹터"란 이 표에 나온 그룹이다.
예: "power_equipment 섹터의 병목 점수" = Eaton + SMCI 평균 3개월 수익률 − AI인프라 전체 평균 3개월 수익률.

### Phase 2~4 — 추가 예정 (3개 섹터)

| 섹터 | 설명 | 예정 기업 |
|------|------|-----------|
| **network** | 데이터센터 내 고속 통신, 스위치 | ANET, AVGO, MRVL |
| **cloud** | 클라우드 플랫폼, AI 서비스 | MSFT, AMZN, GOOGL |
| **datacenter_reit** | 데이터센터 부동산 | EQIX, DLR |

### 파이프라인 순서로 본 섹터 위치

```
[반도체 — semiconductor]
    GPU 설계(NVDA·AMD) → 파운드리(TSM) → 장비(AMAT·LRCX)
          ↓
[전력 장비 — power_equipment]
    서버 전력 관리(ETN·SMCI)
          ↓
[냉각 — cooling]
    액체냉각(VRT) · 열관리(MOD)
          ↓
[전력 유틸리티 — power_utility]
    발전(NEE·CEG·ETR) · 송전망 시공(PWR)
          ↓
(Phase 2 이후)
[네트워크 — network] → [클라우드 — cloud] → [데이터센터 REIT — datacenter_reit]
```

병목은 이 파이프라인 어디서든 발생할 수 있다.
매주 어느 섹터의 점수가 올라가는지 추적하면 "지금 어디서 막혔는가"를 파악할 수 있다.

## 왜 주가가 먼저 움직이는가

병목이 생기면 두 가지 일이 일어난다.

1. **수요자(빅테크)**: "이거 못 구하면 우리 데이터센터 못 짓는다" → 어떻게든 계약한다
2. **공급자(병목 기업)**: 수주잔고 폭증 → 납기 지연 → 가격 협상력 상승

그런데 시장은 이 소식을 어닝콜(분기 실적 발표)에서 공식으로 알게 된다.
어닝콜은 3개월에 한 번이다.

**그 사이에 주가가 먼저 움직인다.**

왜냐하면 "스마트 머니" — 기관 투자자, 업계 관계자, 공급망 분석가들 — 가
어닝콜 전에 수주잔고 증가를 간접적으로 파악하고 먼저 들어오기 때문이다.

이 프로젝트는 그 주가 움직임을 체계적으로 포착하고,
뒤이어 실물 데이터로 확인하는 과정을 만드는 것이다.

## 분석 구조: 3단계 판단 체계

### 1단계: 병목 점수 (Bottleneck Score)

```
병목 점수 = 이 섹터 3개월 수익률 − 전체 AI 인프라 평균 3개월 수익률
```

이 숫자가 크면 "시장이 이 섹터에 돈을 더 많이 넣고 있다"는 뜻이다.

| 점수 | 의미 |
|------|------|
| +10% 초과 | 강한 병목 신호 가능성 |
| +5% ~ +10% | 주목 시작 구간 |
| -5% ~ +5% | 중립 |
| -5% 미만 | 병목 해소 또는 약세 |

**이 점수 하나로 판단하면 안 된다.** 이건 "어디를 봐야 하는가"를 알려주는 탐지기일 뿐이다.

---

### 2단계: 신호 품질 체크 (4가지 질문)

점수가 나왔으면 반드시 아래 4가지를 확인한다.

**Q1. 이 신호가 오르는 추세인가, 꺾이는 추세인가?**
→ 여기서 "점수"는 **병목 점수 v1** = 해당 섹터 3개월 수익률 − 전체 AI인프라 기업 평균 3개월 수익률이다.
→ 예: power_equipment 섹터가 3개월간 +20%, AI인프라 전체 평균이 +8%이면 점수 = **+12%**.
→ 이 점수가 4주 동안 계속 올라가야 유효한 신호다 (예: 4주 전 +5% → 3주 전 +7% → 2주 전 +10% → 이번 주 +12%).
→ 이번 주만 갑자기 높으면 노이즈일 수 있다.

**Q2. 처음 임계값을 넘은 건가, 오래된 신호인가?**
→ +5%를 처음 넘었다면 아직 유효하다.
→ 수개월째 +15%를 유지 중이라면 이미 시장이 다 알고 반영한 상태다.

**Q3. 이 섹터만 올랐나, 전체 시장이 같이 올랐나?**
→ 여기서 "섹터"는 이 프로젝트에서 분류한 기업 그룹이다.
→ 예: power_equipment 섹터 = Eaton(ETN), Vertiv(VRT), Quanta(PWR) 같은 전력 장비 기업들의 묶음.
→ "이 섹터만 올랐나" = "내가 분석 중인 그 섹터 기업들만 올랐나, 아니면 주식시장 전체가 다 올랐나?"
→ SPY(S&P500 전체 시장 ETF)와 비교해서 판단한다.
→ SPY는 제자리인데 이 섹터만 오르면 → 섹터 고유 신호 (신뢰도 높음)
→ SPY도 같이 올랐다면 → 시장 전체가 상승한 거라 섹터 특별 신호가 아닐 수 있음 (주의 필요)

**Q4. 주가 말고 실물 데이터가 있는가?**
→ 수주잔고 증가, 납기 지연 뉴스, 어닝콜 키워드 확인
→ 실물이 뒷받침되면 신뢰도 대폭 상승

---

### 3단계: 최종 판단 매트릭스

1+2단계 결과를 조합해서 8가지 신호 중 하나로 결론낸다.

| 신호 | 의미 | 행동 |
|------|------|------|
| 🔴 강한 병목 신호 | 4가지 체크 모두 통과 | 분할 진입 (1/3씩) |
| 🟡 매크로 랠리 주의 | Q3 실패 (SPY도 상승) | 2주 관찰 후 재확인 |
| ⛔ 고점 경계 | 추세 꺾임, 이미 수개월 지속 | 신규 진입 금지 |
| 🟠 모니터링 강화 | 조건 부분 충족 | 소량 선취매 후 관찰 |
| 🟢 선취매 검토 | 중립이지만 실물 선행 신호 있음 | 소량 진입 |
| ⚪ 중립 | 특이 신호 없음 | 관망 |
| 🔵 섹터 교체 신호 | 약세 + 다른 섹터 강세 | 비중 축소 |
| 🟢 역발상 구간 | 전체 조정 중 펀더멘탈 유지 | 매크로 반등 확인 후 매수 |

## 현재 vs 목표: 지금 뭐가 돌아가고 있나

이 프로젝트는 현재 Phase 1이다.

| 구분 | 상태 | 설명 |
|------|------|------|
| 병목 점수 v1 | ✅ 구현 | 3개월 상대수익률 자동 계산 |
| 신호 품질 체크 Q1~Q3 | ✅ 자동화 가능 | 주가 데이터로 계산 |
| 신호 품질 체크 Q4 | 📋 수동 입력 | 어닝콜 확인은 사람이 해야 함 |
| 공급 제약 지표 (수주잔고 등) | 📋 Phase 2 예정 | SEC EDGAR API 자동 수집 + 어닝콜 수동 보완 |
| 수요 압력 신호 | 📋 Phase 2 예정 | CAPEX 상관관계 분석 |
| 신호 추적 시스템 | ✅ 구현 | signal_log.csv, signal_validation.csv |

**지금 당장 돌아가는 건**: 주가 데이터 기반 병목 점수 자동 계산 + 신호 기록.
**사람이 해야 하는 건**: Q4 (실물 확인) — 어닝콜 텍스트 읽기.

## 실제 사례로 이해하기

### 사례 1: Eaton (ETN) — 전력 변압기 병목

```
시간 순서:
2022년 하반기 — 빅테크 데이터센터 발주 폭증
2023년 초     — 변압기 리드타임 6개월 → 18개월 연장 (어닝콜)
2023년 Q1     — Eaton 수주잔고 25~30% QoQ 증가 (실물 신호)
2023년 중반   — 주가 아웃퍼폼 시작 (가격 신호)
2024년        — 시장 전체가 "전력 인프라 병목"을 인식
```

**이 시스템이 잡으려던 것**: 2023년 Q1 수주잔고 급증 시점.
주가 아웃퍼폼 이전에 실물 신호가 먼저 나왔다.
Q4 체크(실물 확인)를 통해 가격 신호보다 먼저 포착 가능한 구간.

---

### 사례 2: HBM (High Bandwidth Memory) — SK하이닉스

```
시간 순서:
2022년 말     — NVIDIA H100 GPU 수요 폭증
2023년 초     — "H100에는 HBM만 쓸 수 있다" → 공급자 SK하이닉스뿐
2023년 Q1~Q2  — NVIDIA 주가 급등 (가격 신호 상위 레이어)
2023년 하반기 — SK하이닉스 주가 아웃퍼폼 (가격 신호 하위 공급망, 시차 6개월)
```

**이 시스템이 잡으려던 것**: NVIDIA가 터질 때 공급망(HBM)을 같이 보는 것.
상위 레이어 가격 신호 → 하위 공급망 탐색이 이 시스템의 섹터 로테이션 논리.

## 이 시스템의 한계와 솔직한 평가

**현재의 한계**

1. **백테스트 없음**: +5%, +10% 기준값이 최적인지 아직 모른다. 데이터가 쌓여야 검증할 수 있다.
2. **Q4는 수동**: 수주잔고, 어닝콜 확인은 사람이 해야 한다. 완전 자동화는 Phase 3 이후.
3. **한국 주식 미포함**: TSMC 이외의 한국/대만 공급망 기업은 Phase 2 이후.
4. **시차 불확실성**: 실물 신호 → 주가 사이의 시차가 섹터마다 다르다. 정확한 타이밍 예측은 불가능.

**시스템이 잘 작동하는 조건**

- 섹터가 구조적으로 변화하는 중기 트렌드 (6개월~2년 사이클)
- 공급 제약이 실제로 존재하는 경우 (수주잔고, 납기 지연 데이터 있음)
- 매크로 노이즈가 크지 않은 환경

**시스템이 잘 작동하지 않는 조건**

- 금리, 환율, 지정학 이벤트 등 섹터 외부 충격
- 테마 버블 (실물 없이 기대감만으로 오르는 구간)
- 단기 트레이딩 (이 시스템은 중장기용이다)

## 신호 추적 시스템 — 어디서 틀리는지 보는 법

실제로 이 시스템이 맞는지 틀리는지 추적하는 파일이 있다.

```
data/signals/
├── signal_log.csv         ← 주간 신호 기록 (매주 월요일 업데이트)
└── signal_validation.csv  ← 신호 결과 검증 (신호 발생 후 4주, 8주 수익률 입력)
```

### 신호 기록 방법 (매주)

```python
from src.analysis.signal_logger import log_signal

log_signal(
    week_date="2026-03-16",
    sector="power_equipment",
    score_v1=12.3,
    score_trend_4w=3.5,
    weeks_since_breakout=2,
    spy_3m_return=8.2,
    sector_3m_return=28.5,
    avg_3m_return=16.2,
    q4_physical="PASS",  # 어닝콜 확인 후 입력
)
```

### 신호 검증 방법 (4주, 8주 후)

```python
from src.analysis.signal_logger import log_validation

log_validation(
    signal_date="2026-03-16",
    sector="power_equipment",
    original_signal="STRONG_BUY",
    return_4w=6.2,
    return_8w=11.5,
    outcome="CORRECT",
    lesson_tag="GOOD_SIGNAL",
)
```

### 정확도 리뷰 (월 1회)

```python
from src.analysis.signal_logger import print_review

print_review(last_n_weeks=12)  # 최근 3개월 리뷰
```

이 데이터가 쌓이면:
- 어느 섹터에서 신호가 잘 맞는가
- Q1~Q4 중 어느 체크를 놓쳤을 때 틀리는가
- +5%, +10% 기준값이 적절한가

를 데이터로 확인하고 시스템을 개선할 수 있다.
필드 정의 상세는 [docs/signal_log.md](signal_log.md) 참고.

## 인프라 맵 시각화 — 이 프로젝트의 핵심 결과물

### 만들려는 것

Zapier 업무 흐름 도식화처럼, AI 인프라 기업들의 연결 관계를 네트워크 그래프로 표현한다.
여기에 **지금 이 기업이 뜨고 있다**는 것을 노드 크기·색깔로 동적으로 강조한다.

```
[전력공급] ──────→ [반도체/GPU] ←────── [냉각]
 NEE  CEG           NVDA  TSM            VRT  MOD
 ↓ (노드 크기 = 3개월 수익률 강도)
 ↓ (노드 색깔 = 모멘텀 강함(빨강) ~ 약함(파랑))
```

### 구현 방식

| 도구 | 역할 |
|------|------|
| `NetworkX` | 기업 간 연결 관계 정의 (공급망 방향성 포함) |
| `PyVis` 또는 `Plotly` | 인터랙티브 네트워크 그래프 렌더링 |
| `Streamlit` | 웹앱으로 띄우기 — 주간 모멘텀 자동 반영 |
| `yfinance` | 노드 크기·색깔의 데이터 소스 (3개월 수익률) |

### 코드 위치

```
src/visualization/infra_map.py  ← 맵 생성 함수
```
