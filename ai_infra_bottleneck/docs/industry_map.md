## 섹터별 역할 요약

### 반도체 (semiconductor)
- AI 연산의 핵심. GPU 없이는 아무것도 안 된다.
- 설계(NVDA, AMD) vs 파운드리(TSM) vs 장비(AMAT, LRCX) 구분이 중요.
- 현재 병목 강도: **매우 높음** (H100 수요 > 공급)

### 전력 장비 (power_equipment)
- 데이터센터 내 전력 변환·배분 담당.
- 변압기, PDU, UPS, 전력 관리 시스템.
- 납기 지연 심각 → 병목 강도: **높음**

### 전력 유틸리티 (power_utility)
- 전기를 생산하고 송전망을 통해 공급.
- 빅테크의 청정전력 PPA 수요 급증.
- 병목 강도: **높음** (특히 송전망 연결)

### 냉각 (cooling)
- GPU 밀도 증가 → 열 발생량 폭증.
- 기존 공기냉각 → 액체냉각으로 전환 중.
- 병목 강도: **중간~높음** (전환 속도가 변수)

### 네트워크 (network) — Phase 2
- GPU 클러스터 간 고속 통신 (InfiniBand, Ethernet).
- 대규모 모델 학습에서 병목 가능성.

### 클라우드 (cloud) — Phase 2
- AI 인프라의 최종 운영자.
- 수요 창출자이자 공급 통합자.

## 병목 우선순위 (2026년 기준 잠정)

1. **GPU 공급** — 단기 최대 병목
2. **전력망/송전** — 중기 병목 (허가·시공 지연)
3. **전력 장비 납기** — 변압기 대기 2~3년 보고
4. **냉각 전환** — 고밀도 서버 증가로 상승 중
5. **파운드리 생산** — TSMC 증설 속도가 변수

## 인과관계 구조

```
AI 모델 수요 증가
    → 빅테크 GPU 발주 증가 (NVDA 수혜)
    → 데이터센터 건설 증가 (건설·REITs 수혜)
    → 전력 수요 급증 (전력 유틸리티 수혜)
        → 전력망 증설 필요 (Quanta 수혜)
        → 전력 장비 발주 폭증 (Eaton 수혜)
    → 냉각 수요 증가 (Vertiv 수혜)
        → 액체냉각 전환 가속 (Vertiv, Modine 수혜)
```

## 동적 인프라 맵 시각화 계획

### 시각화 아이디어

```
[AI 서비스 수요]
      ↓
[반도체/GPU] ←──── 크기 = 3개월 수익률 강도
 NVDA  AMD         색깔 = 빨강(강한 모멘텀) ~ 파랑(약한 모멘텀)
  TSM
      ↓
[전력 장비]
 ETN  SMCI
      ↓
[냉각]         [전력 유틸리티]
 VRT  MOD       NEE  CEG  ETR  PWR
```

### 기업 간 연결 관계 (엣지 정의)

| 공급자 | 수요자 | 관계 |
|--------|--------|------|
| NVDA | MSFT / AMZN / GOOGL | GPU 공급 |
| TSM | NVDA / AMD | 칩 제조 |
| AMAT / LRCX | TSM | 반도체 장비 |
| ETN | SMCI / 데이터센터 | 전력 장비 |
| VRT | 데이터센터 전체 | 냉각 |
| NEE / CEG | 데이터센터 전체 | 전력 공급 |
| PWR | NEE / CEG | 송전망 시공 |

### 노드 속성 (yfinance 데이터로 자동 업데이트)

| 속성 | 데이터 소스 | 시각화 방법 |
|------|------------|------------|
| 노드 크기 | 3개월 수익률 절댓값 | 클수록 강한 모멘텀 |
| 노드 색깔 | bottleneck_score (섹터 상대 수익률) | 빨강(+) ~ 흰색(0) ~ 파랑(-) |
| 노드 라벨 | ticker + 3개월 수익률 | hover 시 상세 정보 |
| 엣지 두께 | 섹터 간 상관관계 | 두꺼울수록 강한 연동 |
